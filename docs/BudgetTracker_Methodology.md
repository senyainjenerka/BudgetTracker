## 2. Что представляет собой проект BudgetTracker

`BudgetTracker` — это React SPA-приложение на тему финтеха, то есть приложение для отображения и управления финансовыми данными пользователя.

В проекте реализованы:

- отображение списка транзакций;
- отображение категорий бюджета;
- фильтрация по дате, категории и типу транзакции;
- добавление новой транзакции;
- построение графика доходов и расходов;
- генерация текстового отчёта;
- unit-тесты для утилит и ключевых компонентов;
- анализ производительности и качества интерфейса.

По варианту 15 основными объектами являются:

- транзакции;
- бюджетные категории;
- график активности;
- отчёт;
- фильтры;
- форма добавления транзакции.

## 5. Что такое React

React — это библиотека JavaScript для построения пользовательских интерфейсов.

Его идея:

- интерфейс разбивается на маленькие компоненты;
- каждый компонент отвечает за свою часть экрана;
- компоненты можно переиспользовать;
- интерфейс обновляется при изменении состояния.

### 5.1. Почему React удобен

React удобен потому что:

- интерфейс разбивается на части;
- код легче поддерживать;
- данные передаются явно;
- обновления UI происходят предсказуемо;
- удобно тестировать компоненты и отдельные функции.

### 5.2. React — библиотека, а не полноценный фреймворк

React сам по себе отвечает прежде всего за UI.

Он не навязывает полностью всю архитектуру приложения. Для остального обычно подключают:

- роутинг;
- state management;
- тестирование;
- сборку;
- стилизацию;
- работу с API.

В этом смысле React более гибкий, чем многие монолитные фреймворки.

## 6. Что такое SPA

SPA — `Single Page Application`, одностраничное приложение.

Суть SPA:

- HTML-страница загружается один раз;
- дальше контент обновляется через JavaScript;
- пользователь не видит полной перезагрузки страницы при каждом действии.

### 6.1. Как это проявляется в BudgetTracker

В `BudgetTracker`:

- при фильтрации транзакций страница не перезагружается;
- при переключении периода графика обновляется только нужная часть интерфейса;
- при добавлении транзакции не перезагружается вся страница;
- React обновляет только изменившиеся участки интерфейса.

---

## 7. Что такое Virtual DOM

Virtual DOM — это виртуальное представление DOM-дерева в памяти.

Как работает идея:

1. React хранит виртуальную структуру интерфейса;
2. при изменении состояния создаётся новое виртуальное представление;
3. React сравнивает старую и новую версии;
4. обновляет только то, что реально изменилось.

Это позволяет:

- не перерисовывать всю страницу заново;
- снижать лишние обновления;
- делать интерфейс быстрее и удобнее в поддержке.

### 7.1. Почему это важно для твоего проекта

Например, если пользователь:

- меняет фильтр;
- переключает период графика;
- выделяет транзакции;
- добавляет новую транзакцию,

React не перестраивает весь интерфейс полностью, а обновляет только нужные компоненты.

## 9. Что такое JSX

JSX — это расширение синтаксиса JavaScript, которое позволяет писать разметку, похожую на HTML, прямо внутри JavaScript-кода.

Пример:

```jsx
<Header title="BudgetTracker" />
```

Это выглядит как HTML, но на самом деле это не HTML, а специальный синтаксис React.

### 9.1. Зачем нужен JSX

JSX нужен для того, чтобы:

- описывать интерфейс в удобном виде;
- совмещать разметку и логику в одном компоненте;
- делать код компонентов более читаемым.

### 9.2. Чем JSX отличается от HTML

Основные отличия:

- вместо `class` используется `className`;
- JavaScript-выражения пишутся в `{ }`;
- компоненты пишутся с большой буквы;
- обработчики событий передаются как функции;
- атрибуты записываются ближе к синтаксису JavaScript.

Примеры:

```jsx
<div className={styles.card}></div>
```

```jsx
<span>{report.balance}</span>
```

```jsx
<button onClick={handleSelectAll}>Select all</button>
```

## 10. Что такое функциональный компонент

Функциональный компонент — это JavaScript-функция, которая возвращает JSX.

Пример:

```jsx
function Header({ title, subtitle }) {
  return (
    <header>
      <h1>{title}</h1>
      <p>{subtitle}</p>
    </header>
  )
}
```

### 10.1. Почему в проекте используются именно функциональные компоненты

Функциональные компоненты:

- проще писать;
- они короче;
- их легче читать;
- они хорошо сочетаются с React Hooks.

В `BudgetTracker` все основные компоненты сделаны как функциональные.

## 11. Что такое imports в начале файла

В начале React-файлов почти всегда идут `import`.

Они нужны для подключения:

- React и хуков;
- стилей;
- компонентов;
- утилит;
- данных.

Пример из `App.jsx`:

```jsx
import { useEffect, useMemo, useState } from 'react'
```

Это значит:

- взять из библиотеки React три хука:
  - `useEffect`
  - `useMemo`
  - `useState`

Другой пример:

```jsx
import Header from './components/common/Header'
```

Это значит:

- импортировать компонент `Header` из другого файла.

Ещё пример:

```jsx
import {
  calculateBalance,
  generateReport,
  groupTransactionsForChart,
  isWithinDateRange,
} from './utils/budget'
```

Это импорт именованных функций из файла `budget.js`.

## 12. Что такое exports

`export` делает функцию, объект или компонент доступным в других файлах.

Есть два основных вида:

### 12.1. `export default`

Используется, когда файл экспортирует одну основную сущность.

Пример:

```jsx
export default App
```

### 12.2. Именованный export

Пример:

```js

```

Так экспортируются несколько функций из одного файла.

В `budget.js` это удобно, потому что в нём лежит сразу несколько утилит.

## 13. Что такое `main.jsx`

Файл `src/main.jsx` — это точка входа React-приложения.

Он выглядит так:

```jsx
import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'
import './styles/globals.css'
import App from './App.jsx'

createRoot(document.getElementById('root')).render(
  <StrictMode>
    <App />
  </StrictMode>,
)
```

### 13.1. Что здесь происходит

`import { StrictMode } from 'react'`

- подключается `StrictMode`.

`import { createRoot } from 'react-dom/client'`

- подключается функция для создания React-корня.

`import './styles/globals.css'`

- подключаются глобальные стили всего приложения.

`import App from './App.jsx'`

- импортируется главный компонент приложения.

`document.getElementById('root')`

- ищет HTML-элемент с `id="root"` в `index.html`.

`createRoot(...).render(...)`

- говорит React, что нужно отрисовать приложение в этот контейнер.

### 13.2. Что такое `root`

В `index.html` есть:

```html
<div id="root"></div>
```

Это контейнер, в который React “встраивает” всё приложение.

## 14. Что такое `StrictMode`

`StrictMode` — это специальный режим React для разработки.

Он:

- не рисует отдельный интерфейс;
- не добавляет визуальные элементы;
- помогает находить потенциальные ошибки и небезопасные паттерны.

### 14.1. Что именно проверяет `StrictMode`

Он помогает замечать:

- побочные эффекты в неподходящих местах;
- устаревшие практики;
- нестабильный код;
- потенциальные проблемы в логике компонентов.

### 14.2. Почему это важно

В dev-режиме `StrictMode` позволяет раньше находить баги.  
Это полезно, потому что чем раньше замечена проблема, тем легче её исправить.

На защите можно сказать:

> `StrictMode` используется для усиленного контроля качества кода в режиме разработки и помогает находить потенциальные проблемы ещё до production.

## 16. Что такое state

State — это изменяемые данные компонента.

Когда state меняется:

- React запускает новый рендер;
- интерфейс обновляется в соответствии с новым состоянием.

### 16.1. `useState`

`useState` — хук React для хранения состояния.

Пример:

```jsx
const [filters, setFilters] = useState(defaultFilters)
```

Здесь:

- `filters` — текущее значение;
- `setFilters` — функция изменения;
- `defaultFilters` — начальное состояние.

## 17. Что такое `props`

`Props` — это входные данные компонента, которые передаются ему от родительского компонента.

Пример:

```jsx
<Header title="BudgetTracker" subtitle="..." />
```

Компонент получает их так:

```jsx
function Header({ title, subtitle }) {
  ...
}
```

### 17.1. Для чего нужны props

Props нужны для:

- передачи данных вниз по дереву компонентов;
- передачи настроек;
- передачи обработчиков событий;
- повторного использования компонентов в разных контекстах.

### 17.2. Props в твоём проекте

Через props передаются:

- массивы транзакций;
- категории;
- суммы;
- текстовые заголовки;
- выбранный период;
- функции `onChange`, `onToggle`, `onSelectAll`, `onAddTransaction`.

## 18. Что такое `children`

`children` — это специальный prop React, который хранит всё, что вложено внутрь компонента между его открывающим и закрывающим тегами.

Пример:

```jsx
<Card title="Activity">
  <p>Some content</p>
</Card>
```

Здесь `children` — это:

```jsx
<p>Some content</p>
```

### 18.1. Где это используется в проекте

Компонент `Card` — универсальный контейнер.  
Он получает:

- `title`
- `description`
- `children`

Это делает компонент очень удобным:

- обёртка одна;
- содержимое внутри может быть разным.

### 18.2. Почему `children` важны

`children` позволяют создавать переиспользуемые контейнеры и layout-компоненты, не привязывая их к одной жёсткой структуре.

## 19. Что такое hooks

Hooks — это специальные функции React, которые дают функциональным компонентам доступ к возможностям React.

В твоём проекте используются:

- `useState`
- `useMemo`
- `useEffect`

### 19.1. `useState`

Нужен для хранения состояния.

### 19.2. `useMemo`

Нужен для запоминания вычисленного значения.

Он помогает:

- не пересчитывать результат без необходимости;
- повышать производительность;
- не делать тяжёлые вычисления на каждый рендер.

В проекте он используется для:

- `monthTransactions`
- `filteredTransactions`
- `report`
- `visibleSelectedIds`

### 19.3. `useEffect`

Нужен для побочных эффектов.

Побочный эффект — это действие, не являющееся просто вычислением JSX:

- логирование;
- запрос к API;
- подписка на события;
- работа с браузерными API;
- взаимодействие с внешней средой.

В проекте `useEffect` используется для отладочного вывода отчёта в `Console`.

## 23. Бизнес-логика в `budget.js`

Файл `src/utils/budget.js` содержит вынесенную бизнес-логику.

### 23.1. `calculateBalance`

Назначение:

- вычисляет итоговый баланс.

Логика:

- стартует от `openingBalance`;
- проходится по массиву транзакций;
- если транзакция `income`, сумма прибавляется;
- если транзакция `expense`, сумма вычитается.

### 23.2. `categorizeExpense`

Назначение:

- возвращает понятное название категории расхода.

### 23.3. `isWithinDateRange`

Назначение:

- проверяет, попадает ли дата транзакции в выбранный диапазон.

### 23.4. `groupTransactionsForChart`

Назначение:

- группирует транзакции для построения графика.

Режимы:

- `week` — группировка по дням;
- `month` — группировка по недельным сегментам.

### 23.5. `generateReport`

Назначение:

- строит сводный отчёт по выбранным транзакциям.

Что входит в отчёт:

- общее число транзакций;
- количество доходов и расходов;
- общий доход;
- общие расходы;
- чистое изменение;
- итоговый баланс;
- крупнейший расход;
- крупнейший доход;
- разбивка по категориям расходов.

- есть ли `title`;

- корректен ли `robots.txt`;

## 41. Метрики Performance в Lighthouse

### 41.1. FCP — First Contentful Paint

Показывает:

- когда на экране появился первый видимый контент.

Чем меньше, тем лучше.

### 41.2. LCP — Largest Contentful Paint

Показывает:

- когда появился самый крупный видимый элемент страницы.

Это одна из ключевых пользовательских метрик.

### 41.3. TBT — Total Blocking Time

Показывает:

- сколько времени главный поток был занят тяжёлым JavaScript и не мог быстро реагировать на действия пользователя.

Чем меньше, тем лучше.

### 41.4. CLS — Cumulative Layout Shift

Показывает:

- насколько интерфейс “прыгает” при загрузке.

Если пользователь уже хочет нажать кнопку, а блоки внезапно смещаются, `CLS` будет плохим.

Идеальный вариант — значение, близкое к `0`.

### 41.5. Speed Index

Показывает:

- насколько быстро страница визуально наполняется содержимым.

## 46. Что такое breakpoint и как он работает

Breakpoint — это точка остановки выполнения JavaScript-кода.

Когда выполнение доходит до этой строки:

- браузер ставит выполнение “на паузу”;
- можно посмотреть переменные;
- можно идти по коду построчно.

### 46.1. Алгоритм работы breakpoint

1. Открыть `Sources`.
2. Найти файл.
3. Нажать на номер строки.
4. Выполнить действие на странице или обновить её.
5. Когда код дойдёт до этой строки, выполнение остановится.

### 46.2. Что делать после остановки

Можно:

- `Resume` — продолжить выполнение;
- `Step over` — перейти к следующей строке;
- `Step into` — зайти внутрь вызываемой функции;
- `Step out` — выйти из текущей функции.

### 46.3. Что смотреть справа

В панели `Scope` отображаются:

- локальные переменные;
- значения аргументов;
- доступные объекты;
- текущее состояние выполнения.
